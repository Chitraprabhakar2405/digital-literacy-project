# Task 2 – Student Digital Portfolio

## Platforms Set Up
1. **GitHub** – github.com
2. **LinkedIn** – linkedin.com
3. **Kaggle** – kaggle.com

## What Each Platform Is Used For

- **GitHub** is a platform for hosting and sharing code. It acts as a developer's portfolio and is widely checked by recruiters and internship coordinators. I created a profile README with my name, branch, year, and a learning goal.
- **LinkedIn** is a professional networking platform used by companies, recruiters, and academics. I filled in my Education section with my B.Tech degree, VIT Bhopal University, and expected graduation year.
- **Kaggle** is a data science and machine learning platform where students can practise with real datasets, compete in challenges, and build an AI/ML portfolio. As a CSE student, this will be especially useful.

## Reflection 
For this task, I set up professional profiles on GitHub, LinkedIn, and Kaggle. I chose these three platforms because they are the most relevant for a CSE student looking to build a career in technology.

On GitHub, I created a profile README repository and added basic information about myself — my name, branch, year, and a sentence about what I hope to learn: "I hope to learn full-stack web development and machine learning during my four years at VIT Bhopal." This README will evolve as I add projects and skills.

On LinkedIn, I completed the Education section and set my profile to reflect that I am a first-year B.Tech student. I kept everything professional and minimal, as advised.

On Kaggle, I created an account and explored the platform's beginner datasets. I plan to attempt my first notebook challenge in my second semester once I have some Python knowledge.

Over the next four years, I plan to use GitHub to host all my college projects, LinkedIn to connect with alumni and apply for internships, and Kaggle to build a data science portfolio by third year.
